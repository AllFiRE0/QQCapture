# 🏴 QQCapture

**QQCapture** — это продвинутый плагин для организации ивентов по захвату зоны (King of the Hill / Capture the Zone) на серверах Minecraft (Paper/Folia 1.21+). 

Плагин поддерживает несколько зон захвата, начисление очков за тик, систему топ-листов с сохранением в YAML, интеграцию с WorldGuard и PlaceholderAPI.

---

## 📥 Установка

1. Скачайте последний релиз плагина.
2. Поместите файл `.jar` в папку `plugins` вашего сервера.
3. Перезапустите сервер или используйте `/qqcapture reload`.
4. (Опционально) Установите зависимости для расширенной функциональности:
   - **WorldGuard** — для создания/удаления регионов при старте/завершении ивента.
   - **PlaceholderAPI** — для использования заполнителей (Placeholders) в других плагинах.
   - **Vault** — для экономики (если используете баланс в условиях).

## 🎮 Команды

Все команды имеют алиас `/qqc`.

| Команда | Описание | Права |
| :--- | :--- | :--- |
| `/qqcapture start <template> [-p:points] [-d:duration] [-r:region] [-m:multiplier] [-mt:teamMultiplier] [-t:tickCapture] [-s]` | Запускает ивент. | `qqcapture.admin.start` |
| `/qqcapture stop <template>` | Останавливает ивент. | `qqcapture.admin.stop` |
| `/qqcapture list` | Показывает активные ивенты. | `qqcapture.admin.list` |
| `/qqcapture info <template>` | Показывает информацию о шаблоне. | `qqcapture.admin.info` |
| `/qqcapture reload` | Перезагружает конфиги, шаблоны и языковые файлы. | `qqcapture.admin.reload` |
| `/qqcapture top reset <template> clear [-s]` | Очищает топ-данные шаблона. | `qqcapture.admin.top` |
| `/qqcapture top reset <template> <player> reset [-s]` | Удаляет игрока из топа шаблона. | `qqcapture.admin.top` |

### Права (Permissions)
- `qqcapture.participate` — Право участвовать в ивенте (если в шаблоне не указано иное).
- `qqcapture.*` — Все права (только для админов).

## ⚙️ Конфигурация (`config.yml`)

### Основные настройки
| Параметр | Описание |
| :--- | :--- |
| `debug` | Включает подробные логи (рекомендуется для отладки). |
| `language` | Язык плагина (`ru`, `en`, `zh`, `ko`, `ja`). |

### Секция `defaults`
Эти значения применяются, если они не заданы в шаблоне.
| Параметр | Описание |
| :--- | :--- |
| `tick-capture` | Интервал начисления очков (в тиках, 20 тиков = 1 сек). |
| `need-amount` | Целевое количество очков для захвата. |
| `multiplier` | Множитель очков за тик (0.01 = 1% от `need-amount`). |
| `min-players` | Минимум игроков в зоне для начала захвата. |
| `max-players` | Максимум игроков, получающих очки (0 = без лимита). |
| `timer-format` | Формат таймера (`HH:mm:ss`, `mm:ss`, `HH:mm`, `ss`). |
| `send-on-rejoin` | Показывать боссбар при перезаходе игрока. |

## 📝 Шаблоны Ивентов (`templates`)

### 1. Секция `bossbar`
Настройки боссбара и таймингов.
| Параметр | Описание |
| :--- | :--- |
| `enabled` | Включить боссбар. |
| `color` | Цвет (`GREEN`, `YELLOW`, `RED`, `BLUE`). |
| `segments` | Количество сегментов боссбара (1, 6, 10, 12, 20). |
| `update-ticks` | Частота обновления боссбара. |
| `start-delay` / `end-delay` | Задержка появления/исчезновения (сек). |
| `text` | Текст боссбара. Поддерживает заполнители (см. ниже). |

### 2. Игровые настройки
| Параметр | Описание |
| :--- | :--- |
| `need-amount` | Целевые очки (переопределяет `defaults`). |
| `tick-capture` | Интервал начисления (переопределяет `defaults`). |
| `multiplier` | Множитель очков за тик. |
| `type-team-multiplier` | Тип командного бонуса (`individual`, `shared`, `disabled`). |
| `team-multiplier` | Множитель командного бонуса. |
| `min-players` / `max-players` | Лимиты игроков. |
| `permission` | Право для участия (оставьте пустым для всех). |

### 3. Зоны захвата (`zones`)
Вы можете указать **несколько** зон для одного ивента.
```yaml
zones:
  1:
    pos1: "X,Y,Z"
    pos2: "X,Y,Z"
  2:
    pos1: "X,Y,Z"
    pos2: "X,Y,Z"
```

4. Регион WorldGuard (region)

Параметр Описание
region-name Название региона. При наличии создается автоматически.
region-flags Флаги региона (например, {pvp: allow, block-break: deny}).

5. Команды (События)

Плагин поддерживает выполнение команд на разных этапах ивента. Подробнее о синтаксисе смотрите в разделе "Префиксы команд".

Секция Когда выполняется
start-commands Сразу после старта ивента.
tick-commands При каждом начислении очков (тике захвата).
end-commands При завершении ивента.

📊 Заполнители (Placeholders)

Плагин поддерживает собственные заполнители и интеграцию с PlaceholderAPI (используйте %qqcapture_...%).

Общие заполнители

Заполнитель Описание
%qqcapture_current% Текущие очки.
%qqcapture_max% Максимальные очки.
%qqcapture_progress% Прогресс в процентах.
%qqcapture_players% Количество игроков.
%qqcapture_time% Текущее время ивента.
%qqcapture_template% Название шаблона.

Заполнители для конкретного шаблона

Заполнитель Описание
%qqcapture_<template>current% Текущие очки шаблона.
%qqcapture<template>max% Максимальные очки шаблона.
%qqcapture<template>progress% Прогресс шаблона в процентах.
%qqcapture<template>players% Количество игроков в шаблоне.
%qqcapture<template>time% Время ивента шаблона.
%qqcapture<template>template% Название шаблона.
%qqcapture<template>active% Возвращает yes или no.
%qqcapture<template>active<fallback>% Возвращает yes или fallback (с поддержкой цвета).

Заполнитель для игрока

Заполнитель Описание
%qqcapture_player_template% Название шаблона, в котором находится игрок.
%qqcapture_player_template_<fallback>% Название шаблона или fallback (с поддержкой цвета).

Топ-игроки (Top 1-10)

Заполнитель Описание
%qqcapture_<template>top_1_name% Имя победителя.
%qqcapture<template>top_1_value% Очки победителя.
%qqcapture<template>top_2_name% Имя 2-го места.
... ...
%qqcapture<template>_top_10_name% Имя 10-го места.

Участники (Participants)

Заполнитель Описание
%qqcapture_<template>participants<sep>% Список игроков через разделитель.

⚡ Префиксы команд (Синтаксис)

В секциях start-commands, tick-commands и end-commands вы можете использовать следующие префиксы:

1. Команды для игроков

Префикс Описание Пример
message! Сообщение игроку. message! &aПривет!
gMessage! Сообщение всем. gMessage! &aПривет всем!
actionbar! Экшн-бар игроку. actionbar! &aПривет!
gActionbar! Экшн-бар всем. gActionbar! &aПривет всем!
actionbar:X! Экшн-бар игроку на X тиков. actionbar:40! &aПривет!
gActionbar:X! Экшн-бар всем на X тиков. gActionbar:40! &aПривет всем!
title:X:Y:Z! Тайтл игроку (X - появление, Y - показ, Z - исчезновение в тиках). title:20:40:20! Заголовок\|\|Подзаголовок
gTitle:X:Y:Z! Тайтл всем игрокам. gTitle:20:40:20! Заголовок\|\|Подзаголовок
sound! Звук игроку. sound! ENTITY_PLAYER_LEVELUP 1.0 1.0
gSound! Звук всем. gSound! ENTITY_PLAYER_LEVELUP 1.0 1.0

2. Команды от имени

Префикс Описание Пример
asConsole! Команда от консоли. asConsole! say Привет
asPlayer! Команда от игрока. asPlayer! spawn

3. Управляющие команды

Префикс Описание Пример
delay! Задержка (в секундах). delay! 3
check:<условие>! <команда> Выполнить команду при условии. check:%current%>=100000! asConsole! say Done
random:<шанс>! <команда> Случайное выполнение (шанс 0-100). random:50! gMessage! &a50% шанс!

Тайтлы (Title)

Тайтлы поддерживают два разделителя:

· || — всегда разделяет на title и subtitle.
· \n — перенос строки внутри текста (если нужно).

```yaml
# С заголовком и подзаголовком (через ||)
- "title:20:40:20! &6Заголовок||&aПодзаголовок"
- "gTitle:20:40:20! &6Ивент завершен!||&aСпасибо за участие!"

# Только заголовок (без ||)
- "title:20:40:20! &6Тест завершен!"
```

Условия (Check)

Условия поддерживают операторы: >=, <=, ==, !=, >, <, ~ (содержит), !~ (не содержит). Можно объединять условия через && (И) и ~~ (ИЛИ).

```yaml
# Одно условие
- "check:%qqcapture_zona-zahvata_players%>=1! gMessage! &aВ зоне есть игроки!"

# Два условия через И (&&)
- "check:%qqcapture_zona-zahvata_players%>=1 && %qqcapture_zona-zahvata_current%>=100000! gMessage! &aУсловия выполнены!"

# Два условия через ИЛИ (~~)
- "check:%qqcapture_zona-zahvata_players%>=5 ~~ %qqcapture_zona-zahvata_current%>=500000! gMessage! &aОдно из условий выполнено!"

# Вложенные check
- "check:%qqcapture_zona-zahvata_players%>=1! check:%qqcapture_zona-zahvata_current%>=100000! asConsole! say Все условия выполнены!"
```

🛠️ Механики

1. Начисление очков: Очки начисляются каждые tick-capture тиков (по умолчанию 20 тиков = 1 секунда) игрокам, находящимся в зоне.
2. Множители:
   · multiplier: Базовый множитель очков за тик.
   · teamMultiplier: Бонусные очки за нахождение в команде/пати.
3. Топ-листы: По завершению ивента результаты сохраняются в top_data.yml. Игроки могут быть удалены из топа командой /qqcapture top reset.
4. Боссбар: Отображает текущий прогресс захвата, количество игроков в зоне и время.
5. WorldGuard: Автоматически создает регионы по координатам зон при старте ивента и удаляет их после завершения.
6. Fallback (Запасные значения): Все заполнители поддерживают цветной fallback. Если значение отсутствует, отображается указанный текст с поддержкой цветовых кодов &, # и {}.
7. Кэш завершенных сессий: После завершения ивента данные сохраняются в кэш памяти на 60 секунд, чтобы заполнители продолжали работать.
8. Поддержка нескольких зон: Можно указать несколько зон в zones, и плагин будет начислять очки за нахождение в ЛЮБОЙ из них.
9. Автоматическое удаление регионов: При завершении ивента регионы WorldGuard автоматически удаляются.

🌐 ## Интеграции

# PlaceholderAPI
Плагин регистрирует свои заполнители в PlaceholderAPI. Используйте %qqcapture_...% в любых других плагинах (например, в таб-листе, скорборде и т.д.).

# WorldGuard

Если регион указан в шаблоне, плагин автоматически создает его при старте ивента и удаляет при завершении. Флаги региона применяются автоматически.

# Vault

Поддерживается экономика Vault. Используйте %player_vault_balance% в условиях check: для проверки баланса игрока.

```yaml
templates:
  "zona-zahvata":
    bossbar:
      enabled: true
      color: "GREEN"
      text: "Захват зоны: %current%/%max% (Участников: %players%)"
    need-amount: 10000000
    tick-capture: 20
    multiplier: 0.1
    min-players: 1
    max-players: 30
    zones:
      1:
        pos1: "1070,71,-4214"
        pos2: "1058,61,-4224"
      2:
        pos1: "1037,71,-4244"
        pos2: "1051,61,-4232"
    region-name: "capture_zone"
    region-flags: "{pvp: allow, block-break: deny}"
    start-commands:
      - "gMessage! &aИвент захвата зоны начался!"
    tick-commands:
      - "check:%qqcapture_zona-zahvata_current%>=100000! asConsole! eco give %qqcapture_zona-zahvata_top_1_name% 10000"
    end-commands:
      - "gMessage! &6Победитель: &e%qqcapture_zona-zahvata_top_1_name_&7Никто%"
      - "gMessage! &6Очков: &e%qqcapture_zona-zahvata_top_1_value_&c0%"
      - "delay! 3"
      - "gTitle:30:60:30! &6Победа!||&aИвент завершен!"
      - "asConsole! qqcapture top reset zona-zahvata clear -s"
```

❓  FAQ

Вопрос: Почему заполнители показывают ?Никто или 0, хотя я в зоне?
Ответ: Проверьте, запущен ли ивент. Если ивент активен, заполнители берут данные из кэша памяти. Если ивент завершен, они берут данные из top_data.yml. Убедитесь, что файл существует и содержит данные.

Вопрос: Как добавить несколько зон?
Ответ: Добавьте новые зоны в секцию zones с уникальными ID (1, 2, 3...).

Вопрос: Почему регион WorldGuard не создается?
Ответ: Проверьте, что region-name указан в шаблоне и WorldGuard установлен на сервере. Также убедитесь, что зоны имеют корректные координаты.

Вопрос: Как использовать цветной fallback?
Ответ: Добавьте после последнего _ цветовой код, например: _&cНЕТ% для красного текста.

Вопрос: Какие префиксы команд доступны?
Ответ: Все префиксы описаны в разделе "Префиксы команд": message!, gMessage!, asConsole!, asPlayer!, sound!, gSound!, delay!, title:X:Y:Z!, gTitle:X:Y:Z!, actionbar!, gActionbar!, check:, random:.
